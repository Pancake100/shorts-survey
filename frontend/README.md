# Kansei Short-Form Video Survey App v9

This is a static, mobile-first survey app for evaluating short educational videos with Kansei adjective pairs and overall evaluation questions.

## Main features

- Frontend-only static web app
- Mobile-style video screen
- Local MP4 video support
- Group A/B/C video sets
- Randomized video order per participant
- Autogenerated participant ID
- Preliminary questionnaire before the video task
- One multi-part evaluation form per video
- 7-point Kansei adjective scales shown without numeric labels
- Submit final JSON to a configurable API endpoint
- Export JSON backup
- Browser `localStorage` recovery

## Configuration files

### `config/app.config.js`

Edit general app settings:

- experiment title
- experiment version
- submit API endpoint
- participant/session ID prefixes
- general instructions

To enable API submission, set:

```js
submitEndpoint: "https://your-server.example/api/survey-submit"
```

The app sends the final result using:

```http
POST
Content-Type: application/json
```

The submitted JSON includes:

```json
"suggested_filename": "P-YYYYMMDDHHMMSS-RANDOM.json"
```

The server can use this value, or `participant.participant_id`, when saving the result file.

### `config/videos.config.js`

Edit the video groups and local MP4 paths:

```js
{ id: "A01", title: "Video A01", topic: "Science", src: "videos/group-a/a01.mp4" }
```

Put the corresponding MP4 files in:

```text
videos/group-a/
videos/group-b/
videos/group-c/
```

### `config/preliminary.config.js`

Edit the preliminary questionnaire:

- consent
- optional email
- age
- gender
- country/region
- native language
- education level
- short-video watching habits
- educational short-video experience
- topic preference ranking
- language/audio condition

Supported question types include:

```text
email
text
single_choice
multi_choice
rating_5
topic_ranking
```

### `config/evaluation.config.js`

Edit the per-video evaluation form:

- Part A: Visual Impression
- Part B: Content Impression
- Part C: Audio Impression
- Part D: Voice / Narrator Impression
- Part E: Overall Evaluation

Parts A-D use Kansei adjective pairs. The participant sees:

```text
← Attractive                         Unattractive →
○   ○   ○   ○   ○   ○   ○
```

Internally, values are stored as:

```text
-3 -2 -1 0 +1 +2 +3
```

## Participant ID behavior

The participant ID is generated automatically when the setup page is loaded.

Example:

```text
P-20260619143022-X8K2FQ
```

The ID is saved in browser `localStorage`, so refreshing the setup page keeps the same ID. Resetting the survey clears the ID and generates a new one.

## Running locally

For best results, serve the folder through a local web server:

```bash
python3 -m http.server 8000
```

Then open:

```text
http://localhost:8000
```

## Notes on video files

No videos are included in this ZIP. Add your own MP4 files to the video folders and update `config/videos.config.js`.

For research use, make sure the lab has appropriate permission or rights to use and redistribute the video files in the experiment.

## v10 backend-managed group assignment

This version expects the FastAPI backend v2 session flow.

New frontend behavior:

- The participant ID is no longer generated in the browser.
- The video group is no longer manually selected by the participant.
- On a fresh start, the app calls `startSessionEndpoint`.
- The backend returns `participant_id`, `group`, and `group_label`.
- The assigned group is saved in `localStorage` and reused on page reload/resume.
- A URL override such as `?group=B` is passed to the backend only when starting a fresh session.
- Existing saved sessions are not automatically overridden by the URL parameter.

Configure endpoints in `config/app.config.js`:

```js
startSessionEndpoint: "/survey/api/start-session",
submitEndpoint: "/survey/api/submit"
```

For local testing with the backend running directly:

```js
startSessionEndpoint: "http://127.0.0.1:8000/api/start-session",
submitEndpoint: "http://127.0.0.1:8000/api/submit"
```
